LLM Deep Dive

How Large Language Models
Perform Inference

A technical walkthrough of the transformer inference pipeline — from raw text in to generated tokens out.

10 Concepts
1B+ ops per token
1 token per forward pass
01 — Architecture

Decode-Only Transformers

One job: predict the next token given all previous tokens
Each output token is appended and becomes the next input
A deep stack of transformer blocks, each refining what token comes next
Causal masking: tokens can only attend to what came before
02 — Input

Tokenization

Text split into subword chunks — words, pieces, or punctuation
Each chunk maps to an integer ID in a fixed vocabulary
Rare words split into pieces; common words stay whole — "the" = 1 token
Pre-built BPE dictionary — not learned during model training
03 — Representation

Embeddings & Positional Encoding

Token ID → lookup in table → dense vector of ~4096 numbers
Similar tokens cluster nearby: "king" ≈ "queen", "cat" ≈ "dog"
Positional encodings add token order — order changes meaning
Result encodes both what the token is and where it sits
04 — Core Mechanism

Self-Attention

Each token asks: which previous tokens matter most?
Q · Kᵀ computes relevance scores; softmax converts to attention weights
Multi-head attention runs H copies in parallel — grammar, semantics, context
Causal masking: only attend to prior tokens, never ahead
05 — Knowledge Store

Feed-Forward Layers

Two matrix multiplies with activation — most parameters live here
Attention finds relevant context; FFN applies stored knowledge
Acts like a vast compressed database of learned facts and patterns
One attention + one FFN = one block; stacked 32–80× deep
06 — Depth

Layer Stacking

Early layers: syntax, grammar, surface-level features
Middle layers: entities, semantic roles, factual associations
Late layers: task-specific reasoning, next-token prediction
Residual connections pass early information forward — nothing lost
07 — Output

From Vectors to Words

Only the last token's vector matters — it predicts what comes next
Logits are raw scores for every vocabulary token
Softmax converts logits to probabilities; sampling picks the next token
Token appended → sequence grows → loop repeats until done
08 — Optimization

The KV Cache

Without cache: reprocess entire sequence per token — O(n²) cost
KV cache stores K and V; only the new token is computed fresh
Inference becomes memory-bound, not compute-bound
Long contexts are hard: cache grows linearly with sequence length
09 — Inference Phases

Prefill vs. Decode

Prefill: entire prompt processed in one parallel batch
Decode: tokens generated one-at-a-time in a sequential loop
TTFT is prefill speed; tokens/sec is decode throughput
Different hardware and batching strategies target each phase separately
